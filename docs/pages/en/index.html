Silentium = silence

From Latin silentium is translated as silence. The main feature of the silentium library is that an object, if it does not have a good answer to a question, may not give a bad answer, for example - null. This approach allows objects to better control their part of the work, and allows those objects that rely on the work of other objects to trust the received values, without spreading a lot of conditions for checking values ​​throughout the application.

The core of the implementation is that objects are divided into two categories: Sources and Visitors.
Sources are objects with a .value method or functions expecting a visitor in the first argument.
Visitors are objects with a .give method or functions expecting a value in the first argument.

Getting Started

To install the library, use the npm package manager or another one that is more preferable for your application. 

$ npm install silentium

After installing the library, you will be able to use all the developments from the sections
Guest, Patron and Source in your project
