removePatronFromPools function

removePatronFromPools on GitHub

Allows you to remove a patron from all pools. It is useful in cases where the business Logically, the patron should no longer receive updates.

The same effect of removing a cartridge from the pool can be achieved in other ways., for example, if you have access to a bullet pool object, in this case you can call the remove method from the pool

The second way is to use the GuestDisposable class, which accepts 2 the argument. The first argument is the guest, which we turn into a self-canceling guest. The second argument is a function to check whether it is necessary cancel (delete) a patron from pools

The third way is to create your own patron class, implement the GuestDisposableType interface, which describes the disposed method, in cases, when this method returns true, the patron is removed from the pool.
